src.cog.battle module

ワールドチャンネルでの敵とのエンカウントおよびPvE戦闘のUIフローを管理するCog。 定期的に敵をスポーンさせ、ユーザーがそれに対して戦闘を開始できるようにします。

class src.cog.battle.PvE[ソース]

ベースクラス: object

PvE（Player versus Environment）戦闘のフロー全体を管理するクラス。

パーティー選択UIの表示、戦闘の開始、戦闘画面の更新、終了までを一貫して扱います。 このクラス自体は汎用的なフローを定義し、具体的な敵の構成や報酬は `dungeon.py`などのサブクラスで実装されます。

class src.cog.battle.ViewEnemySpawn(pve_obj: PvE, *, timeout: float | None = 180)[ソース]

ベースクラス: View

敵が出現した際に表示されるViewコンポーネント。

「戦う」ボタンを提供し、押下されると戦闘準備フローを開始します。 タイムアウトは1800秒（30分）で、時間内に誰も戦闘を開始しなければ消滅します。

async select_party(int: Interaction, button: Button)[ソース]

パラメータ:

  • button (discord.ui.Button) -- 押されたボタンのインスタンス。
